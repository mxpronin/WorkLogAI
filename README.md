# WorkLog AI

WorkLog AI — offline-first Android-приложение для ежедневного учёта работы ведущего программиста 1С и подготовки данных для СППР. Интерфейс написан на HTML, CSS и JavaScript, Android-сборка выполняется через Capacitor 7.

Отдельный сайт больше не развивается. Корневые веб-файлы остаются исходниками интерфейса Android-приложения и локальной средой проверки перед Capacitor-сборкой.

Текущая версия: `0.6.16`, Android `versionCode 23`.

## Основные сценарии

- календарь является главным экраном; отдельной вкладки «Сегодня» нет;
- задачи содержат номер СППР, название, описание, статус и признак исключения из распределения; его значение фиксируется в новой записи и не меняет созданные ранее записи;
- записи создаются на любую дату и могут содержать текст, голос или до 10 вложений любого типа размером до 25 МБ каждое;
- при добавлении записи активная задача выбирается из полного списка или находится поиском по части номера и названию;
- Android Camera используется для явного действия «Снять фото»;
- голос после остановки автоматически отправляется только на расшифровку, результат сохраняется в редактируемом фактическом тексте;
- улучшение записи запускается отдельно и сохраняется в `spprDescription`, не изменяя исходный текст и реальное время;
- описание СППР сохраняется при изменении времени записи и может быть отредактировано вручную в истории задачи;
- завершение выбранного дня распределяет норму СППР с шагом 30 минут локально или через ИИ;
- для задачи можно задать суммарный максимум СППР-часов; исчерпавшие лимит задачи остаются в реальном учёте, но не получают новое СППР-время;
- общий лимит в настройках применяется ко всем задачам, не уменьшает уже распределённые результаты и при выключении снимает лимиты со всех задач; после применения лимит конкретной задачи можно скорректировать вручную;
- перераспределение выполняется только в контексте одного дня, а не из карточки отдельной задачи;
- единая «История задачи» показывает фактический текст, вариант СППР, реальное и распределённое время каждой записи;
- список задач и история используют одни сохранённые итоги СППР;
- пустой или нулевой день сбрасывается без создания фиктивного результата и может быть начат повторно;
- одновременно может быть начат только один рабочий день;
- перенос и физическое удаление записи сбрасывают только затронутые сохранённые результаты СППР; задача удаляется каскадно вместе с записями, вложениями и ревизиями;
- локальные Android-уведомления приходят в системную строку состояния по включённому интервалу рабочего дня;
- доступны поиск по задачам и работам, подробный экспорт СППР в настоящий XLSX/PDF за произвольный период с дашбордом и системным меню отправки Android, резервная копия и восстановление;
- API-ключ и настройки ИИ сохраняются локально в IndexedDB устройства, ключ исключается из резервной копии и APK.

## ИИ

Поддерживаются OpenAI-совместимые `/responses`, `/chat/completions` и `/audio/transcriptions`. По умолчанию используются:

- текстовая модель: `openai/gpt-5.4-mini`;
- расшифровка: `openai/gpt-4o-mini-transcribe`;
- стандартные промпты версии 5: улучшение записи, улучшение дня и распределение, точная расшифровка, улучшение описания задачи.

Все текстовые ответы ИИ проходят проверку фиксированного JSON-контракта. Поверхностное улучшение только пунктуации отклоняется и один раз отправляется на более содержательную переработку.

## Данные и Android

- IndexedDB schema version: `2`;
- Android application id: `ru.worklogai.app`;
- уведомления: `@capacitor/local-notifications`, канал `worklog-reminders-v2`;
- нативная навигация назад: `@capacitor/app`;
- камера: `@capacitor/camera`;
- временные файлы отчётов и отправка: `@capacitor/filesystem`, `@capacitor/share`;
- JDK: `D:\ProjectApp\jdk-21`;
- Android SDK: `D:\ProjectApp\android-sdk`.

## Проверка

```powershell
cd D:\ProjectApp\sites\worklog-ai
npm test
npx serve .
```

Локальная страница открывается по `http://localhost:3000/`. Она нужна для проверки общих HTML/CSS/JS-исходников, а не как отдельный поддерживаемый сайт.

## Сборка APK

```powershell
cd D:\ProjectApp\sites\worklog-ai
npm install
npm run android:apk
```

Готовый файл:

```text
android\app\build\outputs\apk\debug\WorkLog-AI-0.6.16.apk
```

Команда `android:apk` подготавливает `www/`, выполняет `npx cap sync android` и запускает Gradle. Глобальные `JAVA_HOME`, `ANDROID_HOME` и `ANDROID_SDK_ROOT` не обязательны.

## Структура

- `index.html` — экраны Android-интерфейса;
- `styles/app.css` — mobile-first стили и safe areas;
- `src/app.js` — маршруты, UI-состояние и пользовательские сценарии;
- `src/domain/` — проверяемая бизнес-логика рабочего дня и истории задачи;
- `src/data/indexeddb/database.js` — схема IndexedDB;
- `src/data/repositories.js` — задачи, записи, дни, вложения, настройки и резервные копии;
- `src/ai/` — клиент, промпты, контракты и контроль качества ответа;
- `src/platform/capacitor.js` — доступ к нативным плагинам;
- `tests/` — модульные тесты;
- `scripts/prepare-web.mjs` — подготовка ресурсов Capacitor и cache-buster версии;
- `AGENTS.md` — обязательные продуктовые правила;
- `docs/HANDOFF.md` — подробное актуальное состояние;
- `docs/NEW_CHAT_PROMPT.md` — готовый промпт для продолжения в новом чате.

## Ограничения

- постоянный секундный таймер непосредственно в системном уведомлении потребует отдельного Android foreground-service и сейчас не реализован;
- хранение API-ключа в IndexedDB удобно, но не равно системному защищённому хранилищу;
- камеру, микрофон, реальные уведомления и API после значимых изменений следует дополнительно проверять на Android-устройстве.
