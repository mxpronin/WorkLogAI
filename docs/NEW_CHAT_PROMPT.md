Скопируй весь блок ниже в новый чат Codex.

```text

Это не новый проект. Не создавай демо и не переписывай приложение с нуля. Сначала изучи текущий код и продолжай существующую архитектуру.

Целевой продукт — только Android-приложение на HTML/CSS/JavaScript с упаковкой через Capacitor 7. Отдельный сайт больше не нужен; веб-страница используется только как локальная поверхность проверки общих исходников. Не предлагай React, Flutter, Kotlin/Java или другой стек без моего прямого запроса.

Текущее состояние на 19 июля 2026:
- версия приложения 0.6.17, Android versionCode 24;
- главный экран — календарь, вкладки «Сегодня» нет;
- данные хранятся offline-first в IndexedDB, schema version 2;
- реальные заметки и actualMinutes являются неизменяемыми фактами для ИИ и СППР;
- улучшенный текст хранится отдельно в spprDescription;
- единая «История задачи» показывает исходный текст, текст СППР, реальное и сохранённое СППР-время;
- отдельных вкладок задачи «Реально»/«СППР» и локального перераспределения внутри задачи больше нет;
- распределение СППР выполняется только для выбранного дня с шагом 30 минут;
- задача может иметь суммарный лимит СППР-часов за всё время; общий лимит из настроек применяется ко всем задачам, не уменьшает старые результаты и при выключении снимает лимиты;
- голос сначала только расшифровывается в редактируемый текст, улучшение запускается отдельной кнопкой;
- поиск задачи в форме записи работает по части номера и названию; описание СППР сохраняется при изменении времени и редактируется вручную в истории задачи;
- запись поддерживает до 10 вложений любого типа по 25 МБ и системную камеру Android;
- значение «Не распределять» фиксируется в момент создания записи, перенос между датами инвалидирует затронутые результаты, а записи и задачи удаляются из IndexedDB физически и каскадно;
- одновременно разрешён один активный рабочий день;
- экспорт СППР создаёт XLSX/PDF за выбранный период с дашбордом и открывает системное меню Android для сохранения или отправки;
- Android-уведомления используют @capacitor/local-notifications и канал worklog-reminders-v2;
- стандартные промпты ИИ имеют версию 5 и роли ведущего программиста 1С / ведущего аналитика;
- пользовательские промпты при обновлении сохраняются, старые стандартные мигрируют;
- актуальная подробная модель данных, ограничения и сценарии находятся в docs/HANDOFF.md.

Правила работы:
- сначала проанализируй проблему и найди первопричину, не маскируй ошибки интерфейсным костылём;
- не скрывай дубли в UI, если они реально сохраняются в базе;
- сохраняй совместимость с существующими пользовательскими данными IndexedDB;
- не меняй несвязанные файлы;
- изменения файлов выполняй через apply_patch;
- не добавляй зависимости без практической необходимости;
- после изменений запускай node --check для изменённых JS, npm test и подходящую мобильную визуальную проверку;
- если менялся код приложения, увеличь package version и Android versionCode, затем выполни npm run android:apk;
- перед передачей проверь versionName/versionCode, содержимое Capacitor assets и наличие итогового APK;
- готовый APK называй WorkLog-AI-<version>.apk и давай мне кликабельную ссылку;
- не утверждай, что камера, микрофон, уведомления или реальный API проверены на телефоне, если такой проверки не было;
- задавай вопрос только если отсутствующая информация действительно может заметно изменить решение. В остальных случаях сделай разумное предположение и продолжай.
```
