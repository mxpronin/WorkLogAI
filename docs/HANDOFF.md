# WorkLog AI — актуальное состояние проекта

Дата актуализации: 19 июля 2026

## Назначение

WorkLog AI — личный offline-first рабочий журнал для ведущего программиста 1С. Пользователь фиксирует реальные задачи, заметки и фактическое время, а приложение готовит отдельное описание и распределение времени для СППР.

Целевой продукт — Android-приложение, собранное через Capacitor. Отдельный сайт больше не развивается; корневые HTML/CSS/JS используются как исходники Android-интерфейса и для локальной проверки.

Текущая версия: `0.6.16`, Android `versionCode 23`.

## Реализованные сценарии

### Работа с данными

- CRUD задач, статусы, описание и флаг «Не распределять». Значение флага копируется в новую запись и не меняет старые записи или сохранённые распределения.
- Текстовые и голосовые заметки, а также до 10 локальных вложений любого типа к одной записи (до 25 МБ на файл). Для фото доступна системная камера Android.
- Записи можно добавлять без начала дня; дата выбирается в календаре.
- В форме записи активная задача выбирается из полного списка или находится поиском по части номера и названию.
- Реальное время не изменяется при подготовке СППР.
- При изменении записи сохраняется её готовое `spprDescription`; его можно вручную отредактировать в единой истории задачи без изменения фактической заметки или времени.
- Перед переносом участвующей в СППР записи из завершённого дня приложение предупреждает о сбросе результата. Перенос атомарно обновляет запись и нормализует оба затронутых дня.
- Удаление записи физически удаляет её, все вложения и ревизии одной транзакцией. Удаление задачи остаётся физическим и каскадным.
- Одновременно допускается один начатый рабочий день; старые дубли автоматически переводятся в черновики, кроме наиболее поздно начатого.
- Календарь — главный экран; можно выбрать дату, начать/завершить день, добавить запись и открыть результат.

### СППР и отчёты

- Локальное пропорциональное распределение до нормы дня.
- Исключённые записи не получают СППР-время, но остаются в реальных данных. Решение фиксируется при создании записи по текущему флагу задачи.
- Результат отображается по задачам и записям, виден в календаре, единой истории задачи и отчётах. История задачи одновременно показывает фактический и подготовленный для СППР текст, а также реальное и распределённое время каждой записи; отдельные вкладки «Реально»/«СППР» удалены.
- Экспорт за произвольный период создаёт настоящий однолистовой XLSX и многостраничный PDF с показателями, графиками, рейтингом задач и подробностями. На Android временный файл записывается в `CACHE` и открывается системное меню отправки; в браузере используется скачивание Blob.
- Резервное копирование и импорт IndexedDB включают все локальные вложения.

### ИИ

- OpenAI-совместимые `/responses` и `/chat/completions` через настраиваемый провайдер.
- Четыре режима: улучшение отдельной записи, улучшение дня с распределением, автоматическая расшифровка аудио и улучшение описания задачи.
- После подтверждения перед записью остановка диктофона автоматически запускает только `/audio/transcriptions` и записывает результат в редактируемую фактическую заметку. Профессиональное описание СППР формируется позднее отдельной кнопкой «Улучшить» по сохранённому тексту без повторной отправки аудио.
- Для текста и speech-to-text используются отдельные настраиваемые модели; по умолчанию `openai/gpt-5.4-mini` и `openai/gpt-4o-mini-transcribe`. Для расшифровки доступны `gpt-4o-transcribe` и `gpt-4o-mini-transcribe`; диаризация не используется, поскольку запись рассчитана на одного говорящего.
- Все ответы принимаются только в зафиксированном JSON-формате.
- Стандартные промпты используют роль «Ведущий программист 1С» для записей и дня и роль «Ведущий аналитик и постановщик задач» для описания задачи. Пользовательские шаблоны сохраняются при обновлении; старые стандартные шаблоны автоматически заменяются актуальными. Подсказка speech-to-text остаётся нейтральной и требует дословной расшифровки без редактирования содержания.
- Исходная заметка остаётся фактом, улучшенный текст сохраняется в `spprDescription`.
- Одиночное улучшение получает описание задачи и проверяется на поверхностную корректуру; если первый ответ только меняет пунктуацию, выполняется один повторный запрос на содержательную переработку.
- Если ИИ вернул лишние/повторные задачи, неверную сумму или частичное распределение, приложение отбрасывает неподходящие пункты и нормализует допустимые минуты до нормы дня. При полном сбое распределения используется локальный пропорциональный вариант.
- Ключ API хранится локально в IndexedDB конкретного устройства по решению пользователя; он исключён из резервных копий, исходного кода и APK.
- В настройках ИИ есть отдельный шаблон улучшения описания задачи; кнопка в редакторе задачи заменяет только текст поля описания после явного согласия на отправку.

### Распределение, задачи и уведомления

- СППР-распределение выполняется кратно 30 минутам. Норма рабочего дня также сохраняется с этим шагом, поэтому сумма остаётся точной.
- У задачи есть необязательный суммарный лимит СППР-часов за всё время. Перед распределением из него вычитаются уже сохранённые часы других дней; при исчерпании лимита новые записи остаются только в фактической истории. Старые распределения не изменяются.
- Новая фактическая запись исчерпавшей лимит или исключённой из СППР задачи не инвалидирует готовый результат завершённого дня. Изменение записи, которая действительно входит в сохранённое распределение, по-прежнему сбрасывает результат для честного пересчёта.
- В настройках можно включить общий лимит для всех задач. Он массово задаёт предел, автоматически поднимает его до уже распределённых часов для превысивших значение задач и при выключении снимает лимиты со всех задач. После массового применения лимит конкретной задачи можно изменить вручную.
- Если оставшейся ёмкости всех задач недостаточно для нормы дня, результат СППР остаётся меньше нормы: ограничение имеет приоритет, а интерфейс показывает фактически распределённую сумму.
- Перераспределение запускается только для выбранного рабочего дня. В истории задачи нет локального перераспределения, поскольку изменение одного дня должно сохранять общую норму всех его задач.
- При завершении дня несколько записей одной задачи остаются фактами, но в результате и отчётах отображаются как одна задача с объединённым профессиональным описанием.
- Итоги СППР в списке задач и истории задачи берутся из сохранённых распределений всех дней. Для каждой записи используется сохранённый `entryAllocations`; старый результат без такой детализации не перераспределяется искусственно между несколькими записями.
- Удаление задачи каскадно удаляет связанные записи, вложения и ревизии; результаты затронутых дней сбрасываются в активное состояние для последующего перерасчёта.
- День с заметками до нажатия «Начать день» остаётся черновиком и не показывает «Завершить день»; при запуске мигрируются старые записи `active` без `startedAt`.
- Завершение дня без фактического времени сбрасывает запуск вместо создания пустого результата: день без записей удаляется, день с нулевыми записями возвращается в черновик, после чего его можно начать заново. При запуске также автоматически исправляются пустые результаты, сохранённые предыдущими версиями.
- После «Начать день» приложение запрашивает нативное разрешение и планирует локальные Android-уведомления с выбранным интервалом. Для обычных интервалов покрываются следующие 24 часа; тестовый интервал 2 минуты ограничен 96 ожидающими событиями, чтобы не превышать лимиты Android AlarmManager. Экранный отсчёт появляется только после успешного планирования. Смена интервала перепланирует уведомления активного дня, а завершение дня отменяет ожидающие события.
- Кнопка Android «Назад» возвращает на предыдущий экран внутренней навигации; на корневом экране сворачивает приложение. Свайп влево по карточке задачи открывает «Изменить» и «Удалить».
- В Android resources добавлена обновлённая Material-style иконка WorkLog AI из `assets/worklog-ai-launcher-source.png`; `scripts/generate-launcher-icons.py` создаёт отдельные legacy, round и adaptive foreground ресурсы для всех density.

## Важные технические детали

- IndexedDB schema version: `2`.
- Дата хранится как `YYYY-MM-DD`, длительность — целым количеством минут.
- `workDay.allocations` — отдельный результат СППР; он не меняет записи.
- `workEntry.excludeFromSppr` — снимок флага задачи на момент создания. Отсутствующее поле в старых данных означает участие в СППР.
- `task.maxSpprMinutes` — необязательный суммарный предел задачи; отсутствие поля или `null` сохраняет прежнее распределение без ограничений. Общие настройки хранятся как `globalTaskLimitEnabled` и `globalTaskLimitMinutes`.
- `allocation.entryAllocations` связывает распределённое время с конкретными записями задачи. `src/domain/task-history.js` формирует единую проекцию истории без записи новых значений в базу.
- `settings.aiConfig` содержит провайдера, URL, текстовую модель, модель расшифровки, ключ, версию и пользовательские промпты. При обновлении версии стандартные промпты мигрируют; резервная копия очищает ключ перед экспортом.
- Базовый API URL: `https://openai.api.proxyapi.ru/v1`; модель по умолчанию: `openai/gpt-5.4-mini`.
- Android-уведомления: `@capacitor/local-notifications`; в `capacitor.config.json` указан монохромный значок `ic_stat_worklog`.
- Нативные плагины берутся через `window.Capacitor.Plugins`, который экспортирует Android bridge без сборщика; `registerPlugin` используется как совместимый вариант для окружений с подключённым `@capacitor/core`.
- XLSX создаётся через ExcelJS, PDF через pdfmake с кириллическим Roboto VFS. `@capacitor/filesystem` и `@capacitor/share` используются только для временного Android-экспорта.
- Android application id: `ru.worklogai.app`.
- Командная Android-сборка использует JDK из `D:\ProjectApp\jdk-21`: `android/gradlew.bat` подставляет его только когда `JAVA_HOME` не задан. Android SDK задаётся через `android/local.properties` как `D:\ProjectApp\android-sdk`; глобальные переменные среды не обязательны.
- Для записи голоса Android-манифест содержит `RECORD_AUDIO` и `MODIFY_AUDIO_SETTINGS`; пользователь должен выдать runtime-разрешение при первом запросе. Камера подключена через `@capacitor/camera` и открывается только по явному нажатию «Снять фото».
- Поиск задач включает номер, название, описание и все сохранённые выполненные работы.
- Для точного срабатывания Android-напоминаний добавлено разрешение `SCHEDULE_EXACT_ALARM`.
- Напоминания работают только при включённом переключателе на момент начала дня. Для уведомлений в системной шторке используется новый высокоприоритетный Android-канал `worklog-reminders-v2`; пользователь должен разрешить уведомления и, при запросе Android, точные будильники.
- Цвет нижней системной панели Android установлен в `#080C11`, совпадающий с фоном приложения.
- При подготовке Capacitor-ресурсов `scripts/prepare-web.mjs` автоматически подставляет версию пакета в cache-buster `src/app.js`; Android-сборка не использует устаревший JavaScript из кэша.
- Список записей задачи собирается асинхронно и заменяется в DOM одной операцией, поэтому параллельные вызовы маршрута не создают визуальные дубли карточек.

## Известные ограничения и следующие задачи

1. Браузерные напоминания работают только пока открыта страница. Android-уведомления уже используют Capacitor Local Notifications; живой секундный индикатор именно в системной шторке потребует отдельного foreground-service.
2. Локальное хранение API-ключа в IndexedDB удобно, но не равно защищённому хранилищу. Для production стоит перейти на системное защищённое хранилище или серверный прокси.
3. Голосовую запись и реальный API требуется проверить на Android-устройстве после установки свежего APK: выдать разрешение, подтвердить отправку, начать/остановить запись, дождаться расшифровки, сохранить и воспроизвести запись.
4. Локальная браузерная проверка HTML/CSS/JS не заменяет тестирование нативных разрешений, уведомлений, камеры и микрофона на реальном Android-устройстве.
5. Системное меню отправки XLSX/PDF, сохранение через «Файлы» и отправку в Telegram необходимо финально проверить на реальном Android-устройстве.

## Проверочный сценарий после изменений

1. Открыть календарь, выбрать дату и добавить две записи в разные задачи.
2. Проверить добавление записи из карточки задачи и отсутствие горизонтальной прокрутки на форме.
3. Открыть единую «Историю задачи»: итог СППР должен совпадать со списком задач, а карточка записи должна одновременно показывать реальное и СППР-время. Отдельных вкладок и кнопки перераспределения быть не должно.
4. Запустить голосовую запись, подтвердить разрешение, остановить и сохранить её.
5. Завершить день локально, затем изменить заметку и проверить инвалидирование результата.
6. Завершить день в режиме ИИ; при неидеальном ответе модель не должна ронять приложение.
7. Обновить приложение и проверить данные, API-ключ и пользовательские промпты.
8. Создать резервную копию и убедиться, что ключ API в экспорт не попал.
9. Перенести запись из завершённого дня, отменить предупреждение, затем подтвердить и проверить сброс только затронутых результатов.
10. Открыть экспорт, проверить период по умолчанию, сформировать XLSX и PDF и отправить их через Android Share.
