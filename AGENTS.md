# WorkLog AI: инструкции для продолжения работы

## Область проекта

Этот каталог — самостоятельный продукт WorkLog AI. Все изменения, документация, Android-оболочка и сборочные артефакты продукта остаются внутри `D:\ProjectApp\sites\worklog-ai`.

Корневой `D:\ProjectApp\AGENTS.md` задаёт общие правила рабочего пространства. Этот файл уточняет только WorkLog AI и имеет приоритет для этого каталога, если правила не противоречат корневым.

## Технологии и ограничения

- Чистые HTML, CSS и JavaScript ES-модули; не добавлять React, Flutter, Kotlin/Java-логику или сборщики без отдельного решения пользователя.
- Целевой продукт — Android-приложение. Отдельный сайт больше не развивается; корневые HTML/CSS/JS являются общими исходниками интерфейса и локальной проверочной поверхностью для Capacitor.
- Приложение offline-first, данные — IndexedDB внутри Android WebView.
- Android-оболочка — Capacitor 7, каталог `android/`.
- Изменения файлов выполнять только через `apply_patch`.
- Не добавлять зависимости без явной практической необходимости.
- Не помещать API-ключи в исходники, README, APK или резервные копии. Пользователь явно выбрал локальное сохранение ключа в IndexedDB устройства; не расширять это хранение на экспорт, логи или код.

## Ключевые продуктовые правила

1. Календарь — основной экран. Вкладки «Сегодня» нет.
2. Записи можно добавлять для любой выбранной даты независимо от состояния рабочего дня. «Начать/завершить день» нужны для статуса и будущих уведомлений, не для блокировки ввода. День с записями, но без явного старта, хранится как `draft`; только `active` с `startedAt` позволяет завершение дня.
3. Реальные запись и время — факты. ИИ и СППР никогда не должны перезаписывать `note` или `actualMinutes`.
4. `spprDescription` хранится отдельно. Если оно отсутствует, текст СППР равен исходной заметке.
5. После изменения или удаления записи завершённого дня его распределение становится устаревшим и пересчитывается при следующем завершении.
6. Задачи с `excludeFromSppr: true` сохраняют реальное время, но не участвуют в распределении СППР.
7. Сумма распределения СППР всегда приводится локально к `targetMinutes` дня. Некорректные, лишние и повторные пункты ответа ИИ не должны ронять сценарий.
8. Перед отправкой данных в ИИ требуется явное подтверждение пользователя. Для голоса подтверждение запрашивается перед началом записи, а после остановки автоматически запускается только расшифровка.
9. После голосовой расшифровки её исходный текст сразу записывается в редактируемое поле `note`. Улучшение запускается отдельной кнопкой по сохранённому тексту и записывается в `spprDescription`; аудио повторно не отправляется на расшифровку.
10. Для ProxyAPI идентификаторы моделей сохраняются с префиксом `openai/`. Текстовые GPT-5-модели без суффикса `chat-latest` вызываются через `/responses`, модели `chat-latest` — через `/chat/completions`. Для одного говорящего используются только `gpt-4o-transcribe` и `gpt-4o-mini-transcribe`, без diarize.
11. СППР-время распределяется только с шагом 30 минут; норма дня также приводится к этому шагу. При завершении дня записи одной задачи объединяются в одно описание результата, исходные записи остаются неизменными.
12. Удаление задачи является каскадным: удаляются её записи, локальные вложения любых типов и ревизии, а результаты затронутых завершённых дней сбрасываются для перерасчёта. Это действие требует явного подтверждения.
13. Локальные Android-уведомления реализованы через `@capacitor/local-notifications`: по «Начать день» планируются уведомления на 24 часа строго с выбранным интервалом, а доставленные уведомления не закрываются приложением. Нажатие на уведомление открывает «Новую запись». Тестовый интервал 2 минуты поддерживается; обратный отсчёт до следующего события отображается в приложении. Секундный таймер в системной шторке требует отдельного foreground-service и здесь намеренно не используется.
14. Нативная кнопка Android «Назад» обрабатывается через `@capacitor/app`: она возвращает по внутренней истории экранов, а на корневом экране сворачивает приложение. Карточки задач поддерживают свайп влево для действий «Изменить» и «Удалить».
15. Завершение дня без фактического времени сбрасывает запуск: при полном отсутствии записей рабочий день удаляется, а при наличии нулевых записей возвращается в `draft`, сохраняя сами записи. Пустой результат `finished` не создаётся, пользователь снова видит «Начать день». Поиск задач охватывает номер, название, описание и все тексты выполненных работ (`note`, расшифровку и описание СППР).
16. Для точных Android-напоминаний в манифесте объявлено `SCHEDULE_EXACT_ALARM`; уведомления используют `allowWhileIdle`.
17. Напоминания запускаются только если настройка `remindersEnabled` включена на момент «Начать день». При выключении переключателя локальный таймер, обратный отсчёт и ожидающие Android-уведомления отменяются. Для системной шторки используется канал `worklog-reminders-v2` с высоким приоритетом; внутренний toast не подменяет Android-уведомление.
18. В карточке задачи используется единая «История задачи»: каждая запись показывает фактический и подготовленный для СППР текст, реальное и сохранённое СППР-время. Отдельные вкладки «Реально»/«СППР» и перераспределение внутри задачи не возвращать. Перераспределение допустимо только для выбранного дня, чтобы сохранить его общую норму по всем задачам.
19. Итоги СППР в списке задач, истории задачи, календаре и отчётах должны вычисляться из сохранённых `workDay.allocations`. Не скрывать дубли данных на уровне UI и не создавать новое распределение только ради отображения.
20. Опциональный `task.maxSpprMinutes` ограничивает суммарное СППР-время задачи за все дни. Уже сохранённые распределения не уменьшаются; новые записи получают только оставшуюся ёмкость. Глобальный лимит массово задаёт предел всем задачам, поднимая его до уже распределённого времени там, где оно выше, а при выключении снимает индивидуальные лимиты со всех задач. После массового применения лимит конкретной задачи разрешено корректировать вручную.

## Карта кода

- `index.html` — все экраны и семантическая разметка.
- `styles/app.css` — mobile-first интерфейс; проверять переполнение на узких экранах.
- `src/app.js` — маршруты, UI-состояние, обработчики и прикладные сценарии.
- `src/data/indexeddb/database.js` — схема IndexedDB и транзакции.
- `src/data/repositories.js` — CRUD задач, записей, рабочих дней, настроек, резервных копий и локальных вложений.
- `src/ai/contracts.js` — неизменяемые JSON-контракты и базовые промпты ИИ.
- `src/ai/client.js` — OpenAI-совместимый клиент.
- `src/domain/work-day.js` — правила завершения пустого и нулевого дня.
- `src/domain/task-history.js` — единая проекция реальных записей и сохранённого СППР-времени задачи.
- `src/utils/format.js` — даты, длительности и локальное распределение минут.
- `scripts/prepare-web.mjs` — подготовка `www/` для Capacitor.
- `docs/HANDOFF.md` — актуальное состояние, сценарии и известные ограничения.

## Проверка и сборка

### Локальное Android-окружение

- JDK 21 находится в `D:\ProjectApp\jdk-21`.
- Android SDK находится в `D:\ProjectApp\android-sdk`.
- `android/gradlew.bat` автоматически использует JDK рабочего пространства, если системный `JAVA_HOME` не задан.
- `android/local.properties` фиксирует путь к Android SDK, поэтому глобальные `JAVA_HOME`, `ANDROID_HOME` и `ANDROID_SDK_ROOT` для сборки этого проекта не требуются.
- Полную подготовку веб-ресурсов, Capacitor sync и сборку debug APK запускать одной командой `npm run android:apk` из корня WorkLog AI.
- Готовый APK проверять по пути `android/app/build/outputs/apk/debug/WorkLog-AI-<version>.apk`, включая актуальные `versionCode`, `versionName` и дату файла.

Веб:

```powershell
cd D:\ProjectApp\sites\worklog-ai
npx serve .
```

Открыть `http://localhost:3000/`. После изменения модулей менять query-параметр версии у подключения `src/app.js` в `index.html`, если нужно исключить старый кэш.

Android:

```powershell
cd D:\ProjectApp\sites\worklog-ai
npm run android:apk
```

Перед передачей результата запускать как минимум `node --check` для изменённых JS-модулей, `npx cap sync android` и Android-сборку, если менялись веб-ресурсы или Android-манифест. Не утверждать, что пользовательские клики или запись с микрофона проверены автоматически, если это не выполнялось на реальном устройстве/в браузере.
